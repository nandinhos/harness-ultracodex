# Delivery Format

## Regra central

O formato da entrega deve ser proporcional ao risco e a clareza do pedido.

| Contexto | Formato |
|---|---|
| Duvida pontual | Resposta curta. |
| Sistema ou legado sensivel | Plano tecnico antes de mexer. |
| Risco estrutural | Auditoria. |
| Alvo claro e verificavel | Codigo. |
| Producao sensivel | Execucao incremental. |
| Decisao irreversivel | Confirmacao antes de agir. |

## Red flags

- Entregar codigo quando o problema real e decisao de arquitetura.
- Entregar plano longo para tarefa simples e segura.
- Concluir auditoria sem classificar evidencia.
